Pembuatan Fitur Auth

Disini kita akan mulai proses pembuatan project persewaan kos, kita beri nama Kosan App. Buat project flutter baru dengan perintah

flutter create kosan_app

Selanjutnya anda bisa menghapus platform yang tidak digunakan, misal ios, macos, linux, windows, sisakan hanya android dan web saja.

Struktur Folder Project Flutter​

Kita akan menggunakan struktur folder yang paling mudah dipahami oleh pemula. Intinya masih berprinsip pada separation of concern yakni memisahkan antara tampilan (ui), logika bisnis (business logic), dan data (api).

Berikut adalah struktur folder yang akan kita gunakan:

lib/
│
├── main.dart
│
├── pages/
│   ├── home_page.dart
│   ├── profile_page.dart
│   ├── settings_page.dart
│   └── splash_page.dart
│
├── widgets/
│   ├── custom_button.dart
│   ├── custom_card.dart
│   └── loading_indicator.dart
│
├── models/
│   ├── user.dart
│   ├── product.dart
│   └── api/
│       ├── login_request.dart
│       └── login_response.dart
│
├── providers/
│   ├── auth_provider.dart
│   └── theme_provider.dart
│
├── services/
│   ├── api_service.dart
│   └── storage_service.dart
│
├── utils/
│   ├── constants.dart
│   └── helpers.dart
│
└── config/
    ├── themes.dart
    └── routes.dart

Penjelasan struktur folder:

  1. main.dart: Titik masuk utama aplikasi.

  2. pages/: Berisi semua halaman atau layar dalam aplikasi. Setiap file biasanya berisi satu widget StatelessWidget atau StatefulWidget yang mewakili seluruh layar.

  3. widgets/: Berisi widget yang dapat digunakan kembali di seluruh aplikasi. Ini membantu dalam memecah UI menjadi komponen yang lebih kecil dan dapat dikelola.

  4. models/: Berisi kelas-kelas yang merepresentasikan data dalam aplikasi.

    • Subfolder api/ dapat berisi model khusus untuk request dan response API.

  5. providers/: Berisi provider Riverpod untuk manajemen state. Setiap file biasanya berfokus pada satu fitur atau bagian dari state aplikasi.

  6. services/: Berisi logika bisnis dan interaksi dengan API eksternal atau penyimpanan lokal.

  7. utils/: Berisi fungsi utilitas, konstanta, dan helper yang digunakan di seluruh aplikasi.

  8. config/: Berisi file konfigurasi seperti tema aplikasi dan konfigurasi rute.

Contoh implementasi sederhana untuk beberapa file:

  1. lib/main.dart:
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:go_router/go_router.dart';
import 'config/routes.dart';

void main() {
  runApp(
    ProviderScope(
      child: MyApp(),
    ),
  );
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      title: 'Flutter Demo',
      theme: ThemeData(primarySwatch: Colors.blue),
      routerConfig: goRouter,
    );
  }
}
  1. lib/pages/splash_page.dart:
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:go_router/go_router.dart';
import '../providers/auth_provider.dart';

class SplashPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    ref.listen<AsyncValue<bool>>(
      authProvider,
      (_, state) => state.whenData((isLoggedIn) {
        context.go(isLoggedIn ? '/home' : '/login');
      }),
    );

    return Scaffold(
      body: Center(
        child: CircularProgressIndicator(),
      ),
    );
  }
}
  1. lib/providers/auth_provider.dart:
import 'package:flutter_riverpod/flutter_riverpod.dart';
import '../services/api_service.dart';

final authProvider = FutureProvider<bool>((ref) async {
  final apiService = ref.read(apiServiceProvider);
  return await apiService.checkLoginStatus();
});
  1. lib/config/routes.dart:
import 'package:go_router/go_router.dart';
import '../pages/splash_page.dart';
import '../pages/home_page.dart';
import '../pages/login_page.dart';

final goRouter = GoRouter(
  initialLocation: '/',
  routes: [
    GoRoute(
      path: '/',
      builder: (context, state) => SplashPage(),
    ),
    GoRoute(
      path: '/home',
      builder: (context, state) => HomePage(),
    ),
    GoRoute(
      path: '/login',
      builder: (context, state) => LoginPage(),
    ),
  ],
);

Struktur ini memberikan organisasi yang jelas dan mudah dipahami, sambil tetap memisahkan berbagai aspek aplikasi (UI, logika bisnis, data, konfigurasi). Ini membuat kode lebih mudah dikelola dan diperluas seiring pertumbuhan aplikasi.
